# Spec — F-DC-003: Preview before import

**Version:** v1
**Status:** FROZEN
**Last updated:** 2026-05-02
**Pillar:** pillar-01-data-connectors
**Feature ref:** F-DC-003
**Dependencies:** F-DC-002

## Summary
Before committing data to the database, show first 50-100 rows with detected schema. User confirms or adjusts column names/types, chooses a table name, then clicks Import.

## Inputs
| Field | Type | Required | Default | Validation |
|---|---|---|---|---|
| table_name | string | yes | filename (sanitized) | alphanumeric + underscores, max 64 chars, unique in DB |
| columns | array | yes | from F-DC-002 | at least 1 column |

## States

### previewing
Table preview visible: first 50 rows, column headers with types. Table name input field at top. Editable column names. "Import" and "Cancel" buttons.

### importing
User clicked Import. Data being written to database. Spinner: "Creating table…"

### success
Table created. Message: "Table '{{table_name}}' created with {{row_count}} rows and {{col_count}} columns." Link to view in catalog.

### error:name-conflict
Table name already exists in DB.
Message: "Table '{{table_name}}' already exists. Choose a different name or overwrite."
Options: rename field + "Overwrite existing" checkbox.

### error:import-failed
DB write failed (disk full, connection error).
Message: "Import failed: {{error}}. Try again."

## Transitions
| From | To | Trigger | Side effects |
|---|---|---|---|
| previewing | importing | user clicks Import | begin DB write |
| previewing | (back to empty) | user clicks Cancel | temp data cleared |
| importing | success | DB write completes | table created, auto-registered in catalog (F-SC-003) |
| importing | error:name-conflict | table name exists | none |
| importing | error:import-failed | DB error | none |
| error:name-conflict | importing | user renames or checks overwrite | retry with new name |

## Edge cases
- User renames column to empty string → validation error: "Column name cannot be empty"
- User renames two columns to same name → validation error: "Duplicate column name"
- Table name with spaces → auto-replace with underscores
- Very wide preview (100+ columns) → horizontal scroll
- 0-row import (headers only) → allow, show warning: "Table will be empty (0 rows)"

## Acceptance checklist
- [ ] Preview shows correct data and types → user clicks Import → table created
- [ ] User changes table name → new name used
- [ ] User renames a column → new name in DB
- [ ] Existing table name → error:name-conflict with overwrite option
- [ ] Cancel → returns to upload area, no table created
