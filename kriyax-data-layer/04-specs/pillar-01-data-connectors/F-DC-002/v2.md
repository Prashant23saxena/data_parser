# Spec — F-DC-002: Auto-detect columns & types

**Version:** v2
**Status:** FROZEN
**Last updated:** 2026-05-02
**Pillar:** pillar-01-data-connectors
**Feature ref:** F-DC-002
**Dependencies:** F-DC-001

## Summary
After file upload, parse headers and sample rows to infer column names and data types (string, integer, float, date, boolean). Handle CSV quirks (delimiters, encoding, quoted fields).

## Inputs
| Field | Type | Required | Default | Validation |
|---|---|---|---|---|
| parsed_file | internal | yes | — | from F-DC-001 success |
| sample_rows | int | no | 100 | 10-1000 |

## States

### detecting
System scanning headers and sampling rows. Spinner: "Detecting columns and types…"

### detected
Column list with inferred types ready. Shows table: column name, detected type, and sample values. Types are shown for review but are not manually overridden in v1.

### needs-rename
Duplicate column names are detected. User must rename duplicate columns before continuing. Message: "Duplicate column names found. Rename the highlighted columns to continue."

### error:no-columns
File has no parseable columns (e.g., single empty column).
Message: "Could not detect any columns. Check the file format."

## Transitions
| From | To | Trigger | Side effects |
|---|---|---|---|
| detecting | detected | parsing complete with unique column names | column list + types stored |
| detecting | needs-rename | duplicate column names detected | duplicate columns highlighted |
| detecting | error:no-columns | 0 columns found | none |
| needs-rename | detected | user provides unique names for duplicates | renamed column list stored |
| detected | (hand off to F-DC-003) | user confirms detected columns and types | detected schema stored |

## Edge cases
- Column names with spaces/special chars → preserve as-is, sanitize only for DB column names later
- Duplicate column names → ask user to rename duplicates before continuing; do not auto-suffix silently
- Mixed types in column (some rows int, some string) → default to string
- Date format ambiguity (01/02/2024 = Jan 2 or Feb 1?) → default to locale-aware and show detected format for review
- Very wide files (100+ columns) → show scrollable table, no limit

## Acceptance checklist
- [ ] CSV with clear headers and types → types detected correctly
- [ ] CSV with mixed int/string column → defaults to string
- [ ] Excel with dates → date type detected
- [ ] Detected types are shown for review without manual override controls
- [ ] File with duplicate column names → user is asked to rename duplicates before continuing
