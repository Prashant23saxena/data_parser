## Summary
Platform helper function `load_table(table_name)` available in script execution context. Reads a database table and returns a pandas DataFrame.

## Inputs
| Field | Type | Required | Default | Validation |
|---|---|---|---|---|
| table_name | string | yes | — | must exist in DB |

## Behavior
```python
df = load_table("sale_orders")
# Returns: pandas DataFrame with all rows and columns
# Types: mapped from DB types to pandas types
```

## States (runtime, not UI)

### success
DataFrame returned with correct data and types.

### error:table-not-found
Table doesn't exist. Raises: `ValueError: Table 'xxx' not found. Available tables: [list]`

### error:db-connection
DB connection failed. Raises: `ConnectionError: Cannot connect to database`

## Edge cases
- Table with 0 rows → returns empty DataFrame with correct columns
- Table with 1M+ rows → loads all by default (warn in docs about memory)
- Column names with spaces → accessible via `df["column name"]`
- load_table called twice for same table → no caching, fresh read each time

## Acceptance checklist
- [ ] `load_table("existing_table")` → DataFrame with correct data
- [ ] `load_table("nonexistent")` → ValueError with available tables listed
- [ ] Empty table → DataFrame with columns but 0 rows
- [ ] Types preserved (dates are datetime, ints are int64)
