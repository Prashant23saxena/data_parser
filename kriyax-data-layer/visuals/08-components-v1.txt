  |-- uses --> backend-workspace-service
  |-- reads live metadata from --> DuckDB Warehouse
  |-- stores app metadata in --> workspace metadata JSON

Dependency rules:
  - No frontend component reads files or DuckDB directly.
  - API routers stay thin.
  - Domain services own their entities.
  - Pipelines orchestrate; execution and connector sync remain delegated.
  - No circular service dependencies in the target architecture.
