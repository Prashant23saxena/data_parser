## Features

### F-CW-001: Python code editor

**Description:** A code editor where the user writes Python/pandas code in a single file. Supports syntax highlighting and a "Run" button to execute the code against the platform's data layer.

**Priority:** must

**Dependencies:** none

**Definition of done:** User can write Python code in an editor, click Run, and the code executes in a sandboxed Python environment with pandas and database access pre-configured.

**Cross-pillar links:** none

---

### F-CW-002: Show available tables

**Description:** A sidebar or panel next to the editor that lists all tables from the Schema & Catalog (Pillar 2). Shows table names and column info so the user knows what data is available before writing code.

**Priority:** must

**Dependencies:** F-SC-001 (Schema & Catalog: Table registry)

**Definition of done:** User sees a list of all available tables alongside the code editor. Clicking a table shows its columns and types. List stays in sync with the catalog.

**Cross-pillar links:** Pillar 2 (Schema & Catalog) — reads table registry.

---

### F-CW-003: Load tables as DataFrames

**Description:** A simple helper/utility so the user can load any database table into a pandas DataFrame with one line of code (e.g., `df = load_table("my_table")`). Pre-configured database connection — no manual setup.

**Priority:** must

**Dependencies:** F-CW-001

**Definition of done:** User calls `load_table("table_name")` in their Python code and gets a pandas DataFrame with all rows and columns from that table. Connection is pre-configured, no credentials needed in user code.

**Cross-pillar links:** Pillar 6 (Data Storage) — reads from backing database.

---

### F-CW-004: Save DataFrame as table

**Description:** A helper to write a pandas DataFrame result back to the database as a new or updated table (e.g., `save_table(df, "cleaned_orders")`). Creates the table if it doesn't exist, replaces data if it does.

**Priority:** must

**Dependencies:** F-CW-003

**Definition of done:** User calls `save_table(df, "name")` and the DataFrame is persisted as a database table. Table appears in the Schema & Catalog immediately. Supports create-or-replace behavior.

**Cross-pillar links:** Pillar 6 (Data Storage) — writes to backing database. Pillar 2 (Schema & Catalog) — new table auto-registers via F-SC-003.

---

### F-CW-005: View execution output

**Description:** After running code, the user sees the output below the editor — print statements, DataFrame previews (first/last rows), error messages with tracebacks. Clear visual distinction between success and error output.

**Priority:** must

**Dependencies:** F-CW-001

**Definition of done:** User clicks Run, output panel shows stdout, DataFrame `.head()` previews, and any errors with full tracebacks. Output clears on re-run.

**Cross-pillar links:** none

---

### F-CW-006: Save & re-run scripts

**Description:** User saves their Python script with a name (e.g., "clean_orders_pipeline"). Saved scripts are listed for quick access. Any saved script can be re-run on demand with one click. Saved scripts are also available for Pillar 5 (scheduling).

**Priority:** must

**Dependencies:** F-CW-001

**Definition of done:** User can save the current editor content with a name, see a list of all saved scripts, open any saved script in the editor, and re-run it. Scripts persist across sessions.

**Cross-pillar links:** Pillar 5 (Pipeline & Scheduling) — saved scripts are the units that get scheduled for automated runs.
