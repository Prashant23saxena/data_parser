## Features

### F-AL-001: Chat interface

**Description:** A chat panel (sidebar or bottom panel) where the user types plain English requests like "merge orders with customers and show total revenue per region". The agent responds with generated Python code and brief explanation.

**Priority:** must

**Dependencies:** none

**Definition of done:** User can type a plain English data request in a chat input, submit it, and receive a response from the LLM agent. Chat history is visible in the panel.

**Cross-pillar links:** none

---

### F-AL-002: Schema-aware context loading

**Description:** Before generating code, the agent reads table and column metadata directly from the database (information_schema or equivalent). It knows what tables exist, their column names, and data types — so generated code references real table/column names.

**Priority:** must

**Dependencies:** F-SC-001 (Schema & Catalog: Table registry)

**Definition of done:** Agent's LLM prompt includes current table names and column metadata. If user says "show me all orders", the agent knows the table is called `odoo_sale_order` (not a guess). Metadata is loaded fresh on each request.

**Cross-pillar links:** Pillar 2 (Schema & Catalog) — reads table/column metadata from DB.

---

### F-AL-003: Generate Python/pandas code

**Description:** Agent produces runnable Python/pandas code based on the user's English request plus the schema context. Code uses the platform helpers (`load_table()`, `save_table()`) from Pillar 3. All transformations are in pandas — no raw SQL generation.

**Priority:** must

**Dependencies:** F-AL-001, F-AL-002

**Definition of done:** Agent generates syntactically correct Python/pandas code that uses `load_table()` to read data, performs the requested transformation (filter, join, aggregate, clean), and optionally uses `save_table()` to persist results. Code is ready to run as-is in the Python editor.

**Cross-pillar links:** Pillar 3 (Code Workspace) — generated code uses Pillar 3's helpers and runs in its editor.

---

### F-AL-004: Insert code into editor

**Description:** When the agent generates code, user can click "Insert into editor" to push the code directly into the Python code editor (Pillar 3). User reviews, optionally edits, then runs it.

**Priority:** must

**Dependencies:** F-AL-003, F-CW-001 (Code Workspace: Python code editor)

**Definition of done:** Generated code appears in the Pillar 3 editor with one click. User can edit before running. Does not auto-execute — user must explicitly click Run.

**Cross-pillar links:** Pillar 3 (Code Workspace) — writes into the code editor.

---

### F-AL-005: Self-correction on error

**Description:** If the user runs the generated code and it fails (runtime error, wrong column name, etc.), the agent reads the error traceback and generates a corrected version. User can click "Fix this" to trigger the correction cycle.

**Priority:** should

**Dependencies:** F-AL-003

**Definition of done:** When a script errors out, user can send the error to the agent. Agent reads the traceback, identifies the issue, and generates corrected Python code. Works for common errors: KeyError (wrong column), TypeError, missing data, etc.

**Cross-pillar links:** Pillar 3 (Code Workspace) — reads error output from execution.

---

### F-AL-006: Conversational follow-ups

**Description:** User can refine previous requests in the same chat session: "now filter this by date > 2024", "add a column for profit margin", "group by region instead". Agent builds on the previously generated code rather than starting from scratch.

**Priority:** should

**Dependencies:** F-AL-003

**Definition of done:** Agent maintains conversation context within a session. Follow-up requests produce code that extends or modifies the previous code rather than regenerating from zero. Context resets when user starts a new chat or script.

**Cross-pillar links:** none
