# Integration Tests — KriyaX Data Layer

**Version:** v1
**Status:** FROZEN
**Last updated:** 2026-05-03
**Refs:** L8.3 contracts v1, L8.5 work packages v1

## Overview

Eight integration scenarios verify every frozen contract and the cross-component wires for the remaining build.

## Test Environment

- Backend: FastAPI test client or live local backend on `127.0.0.1:8001`.
- Frontend: Playwright against Vite dev server where UI is involved.
- Warehouse: temp `KRIYAX_WORKSPACE_ROOT` per test run.
- External services: Odoo and LLM provider mocked unless a sandbox is explicitly configured.

## Scenarios

### INT-001: File Import Registers Catalog Table

**Components:** frontend-connectors, backend-imports-service, backend-storage-service, backend-catalog-service.
**Contracts:** C-001, C-003, C-004.
**WP preconditions:** existing baseline.

Steps:
1. Upload CSV through imports API or UI.
2. Commit to `raw_files.orders`.
3. Load catalog table list.
4. Preview table rows.

Expected:
- Catalog contains the table.
- Preview returns rows and columns.
- Duplicate column error follows C-004.

### INT-002: Python Execution Creates Derived Catalog Table

**Components:** frontend-code-workspace, backend-execution-service, backend-catalog-service, backend-storage-service.
**Contracts:** C-001, C-003, C-005.
**WP preconditions:** existing baseline.

Steps:
1. Ensure an input table exists.
2. Run script with `load_table()` and `save_table()`.
3. Check execution response.
4. Check derived table appears in catalog.

Expected:
- `savedTables` includes the derived table.
- Catalog row source is `python`.

### INT-003: Odoo Fetch Registers Odoo Table

**Components:** frontend-connectors, backend-odoo-service, backend-catalog-service, backend-storage-service.
**Contracts:** C-001, C-003, C-006.
**WP preconditions:** existing Phase 2 baseline.

Steps:
1. Mock Odoo connection success.
2. Browse models and fields.
3. Fetch selected fields into table.
4. Check catalog registration.

Expected:
- Unsupported fields produce warnings.
- many2one fields flatten into ID and display name.

### INT-004: Scheduled Pipeline Runs and Records History

**Components:** backend-pipelines-service, backend-execution-service, frontend-pipelines.
**Contracts:** C-005, C-007.
**WP preconditions:** WP-B01, WP-B02.

Steps:
1. Create pipeline from saved script.
2. Save daily schedule.
3. Restore scheduler.
4. Trigger due schedule.
5. Load run history and detail.

Expected:
- Scheduled run has trigger `schedule`.
- `nextRunAt` recalculates.
- Disabled pipeline skips automatic run.

### INT-005: Failed Pipeline Surfaces Notification

**Components:** backend-pipelines-service, frontend-pipelines, frontend-app-shell.
**Contracts:** C-007.
**WP preconditions:** WP-B03, WP-F01.

Steps:
1. Run or schedule a pipeline with failing script.
2. Load failures endpoint.
3. View Home and Pipelines.
4. Acknowledge failure.

Expected:
- Active failure appears.
- Ack hides active alert but keeps history.
- New failure reopens alert.

### INT-006: Odoo Sync Pre-Step Runs Before Script

**Components:** backend-pipelines-service, backend-odoo-service, backend-execution-service.
**Contracts:** C-006, C-007, C-005.
**WP preconditions:** WP-C01, WP-C02.

Steps:
1. Configure sync cursor.
2. Create pipeline with `connectorSyncId`.
3. Run pipeline.
4. Inspect run detail.

Expected:
- Sync step runs before script.
- Sync failure blocks script and records error.
- Successful sync then executes script.

### INT-007: Table Management Updates Catalog

**Components:** backend-storage-service, backend-catalog-service, frontend-catalog.
**Contracts:** C-002, C-003.
**WP preconditions:** WP-D01, WP-D02, WP-F01.

Steps:
1. Rename a table.
2. Truncate a table.
3. Export a table.
4. Drop a table.

Expected:
- Catalog reflects every mutation.
- Export returns CSV.
- Protected/internal tables cannot be modified.

### INT-008: Agent Generates and Corrects Code

**Components:** backend-agent-service, backend-catalog-service, backend-execution-service, frontend-agent-panel, frontend-code-workspace.
**Contracts:** C-003, C-005, C-008.
**WP preconditions:** WP-E01, WP-E02.

Steps:
1. Ask agent to create pandas code from available catalog table.
2. Insert code into editor.
3. Run failing code.
4. Send stderr for correction.

Expected:
- Agent uses real schema names.
- Code uses `load_table()` and `save_table()`.
- Corrected code is proposed, not auto-run.
- Sample rows are excluded by default.
