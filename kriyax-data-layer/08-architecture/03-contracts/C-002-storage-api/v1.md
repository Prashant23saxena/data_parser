# Contract C-002: Storage API

**Version:** v1
**Status:** FROZEN
**Last updated:** 2026-05-03
**Type:** http-api
**Producer:** backend-storage-service via backend-api-routers
**Consumers:** frontend-storage-settings, frontend-catalog

## Purpose

Expose local workspace and DuckDB health plus table management/export operations.

## Versioning Policy

Breaking endpoint path, method, field, or status-code changes require v2. Additive optional fields may remain v1.

## Shape

### GET `/api/storage/status`

Response 200:
```json
{
  "status": "ready",
  "databaseReady": true,
  "databasePath": "/workspace/warehouse/kriyax.duckdb",
  "workspaceRoot": "/workspace",
  "checkedAt": "2026-05-03T00:00:00+00:00",
  "error": null
}
```

### POST `/api/storage/tables/{schema}/{tableName}/rename` target

Request:
```json
{"newTableName":"clean_orders"}
```

Response 200: `CatalogTable`

### POST `/api/storage/tables/{schema}/{tableName}/truncate` target

Request:
```json
{"confirmation":"schema.tableName"}
```

Response 200:
```json
{"qualifiedName":"raw.orders","rowCount":0}
```

### DELETE `/api/storage/tables/{schema}/{tableName}` target

Request body:
```json
{"confirmation":"schema.tableName"}
```

Response 200:
```json
{"deleted":true,"qualifiedName":"raw.orders"}
```

### GET `/api/storage/tables/{schema}/{tableName}/export.csv` target

Response 200: `text/csv` download stream.

Errors:
- 400 `validation_failed`
- 404 `table_not_found`
- 409 `protected_table`
- 500 `storage_unavailable`

## Examples

### Happy Path

GET storage status returns `databaseReady: true` and S-14 shows ready.

### Error

Drop without exact confirmation returns:
```json
{"error_code":"validation_failed","message":"Type the qualified table name to confirm.","fields":{"confirmation":"mismatch"}}
```

### Edge Case

Exporting a 0-row table returns a CSV with headers only.

## Mock Specification

Mocks return ready status by default. For table actions, return success when `confirmation` equals `{schema}.{tableName}`; return 400 otherwise. Export mock returns two-line CSV.
