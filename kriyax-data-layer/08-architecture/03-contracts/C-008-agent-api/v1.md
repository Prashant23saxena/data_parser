# Contract C-008: Agent API

**Version:** v1
**Status:** FROZEN
**Last updated:** 2026-05-03
**Type:** http-api
**Producer:** backend-agent-service via backend-api-routers
**Consumers:** frontend-agent-panel, frontend-code-workspace

## Purpose

Generate Python/pandas code from natural language using current catalog metadata, support follow-ups, and propose fixes from execution errors without auto-running code.

## Shape

### POST `/api/agent/generate`

Request:
```json
{"conversationId":null,"prompt":"Join orders with customers and save revenue by region","currentCode":"","includeSampleRows":false}
```

Response 200:
```json
{"conversationId":"conv123","messageId":"msg123","code":"orders = load_table('raw_files.orders')\n# ...","explanation":"Loads available tables and groups revenue by region.","schemaContext":{"tables":["raw_files.orders"]}}
```

### POST `/api/agent/correct`

Request:
```json
{"conversationId":"conv123","code":"df['missing']","stderr":"KeyError: 'missing'","includeSampleRows":false}
```

Response 200:
```json
{"conversationId":"conv123","messageId":"msg124","code":"df['existing_column']","explanation":"The column name was corrected using current catalog metadata.","attemptNumber":1}
```

### POST `/api/agent/follow-up`

Request:
```json
{"conversationId":"conv123","prompt":"Now filter this for 2024","currentCode":"orders = load_table('raw_files.orders')"}
```

Response 200: same as generate.

Errors:
- 400 `validation_failed`, `ambiguous_request`
- 404 `conversation_not_found`
- 429 `llm_rate_limited`
- 500 `generation_failed`

## Examples

### Happy Path

Prompt asks for a join. Response code uses only `load_table()` and `save_table()` helpers and is inserted into editor only after user action.

### Error

Ambiguous table reference returns:
```json
{"error_code":"ambiguous_request","message":"Multiple order tables exist. Choose one: raw_files.orders, odoo.sale_order."}
```

### Edge Case

Second correction failure should stop and ask user to inspect manually; no infinite correction loop.

## Mock Specification

Generate mock returns deterministic code using the first catalog table. Correct mock changes `missing_column` to `customer` if stderr includes `KeyError`. If prompt contains `ambiguous`, return 400.
