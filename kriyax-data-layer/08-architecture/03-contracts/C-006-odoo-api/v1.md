# Contract C-006: Odoo API

**Version:** v1
**Status:** FROZEN
**Last updated:** 2026-05-03
**Type:** http-api
**Producer:** backend-odoo-service via backend-api-routers
**Consumers:** frontend-connectors, frontend-pipelines

## Purpose

Configure Odoo connections, browse Odoo metadata, fetch selected records to DuckDB, and provide the target connector sync operation for pipelines.

## Shape

### POST `/api/odoo/test-connection`

Request:
```json
{"connectionName":"Client Odoo","odooUrl":"https://client.odoo.com","dbName":"client","username":"user@example.com","apiKey":"secret"}
```

Response 200:
```json
{"status":"success","message":"Connected successfully","odooUrl":"https://client.odoo.com","version":"17.0","uid":12}
```

### POST `/api/odoo/connections`

Response 200: same connection shape with masked `apiKey`.

### GET `/api/odoo/connections`

Response 200:
```json
{"connections":[{"connectionName":"Client Odoo","odooUrl":"https://client.odoo.com","dbName":"client","username":"user@example.com","apiKey":"********","createdAt":"2026-05-03T00:00:00+00:00"}]}
```

### POST `/api/odoo/models?search=sale`

Request: Odoo connection payload.

Response 200:
```json
{"models":[{"model":"sale.order","name":"Sales Order","fieldCount":120}]}
```

### POST `/api/odoo/models/{modelName}/fields`

Response 200:
```json
{"fields":[{"name":"partner_id","label":"Customer","type":"many2one","required":false,"readonly":false,"relation":"res.partner","supported":true}]}
```

### POST `/api/odoo/fetch`

Request:
```json
{"connection":{"connectionName":"Client Odoo","odooUrl":"https://client.odoo.com","dbName":"client","username":"user@example.com","apiKey":"secret"},"modelName":"sale.order","fields":["id","partner_id"],"targetSchema":"odoo","tableName":"sale_order","domainFilter":[],"recordLimit":1000}
```

Response 200: `CatalogTable`

### POST `/api/odoo/sync/{syncCursorId}` target

Response 200:
```json
{"status":"success","targetQualifiedName":"odoo.sale_order","inserted":4,"updated":8,"lastCursorValue":"2026-05-03T00:00:00+00:00"}
```

Errors:
- 400 `validation_failed`, `invalid_domain`, `unsupported_fields`
- 401 `auth_failed`
- 404 `connection_not_found`, `model_not_found`
- 409 `table_exists`
- 503 `odoo_unreachable`

## Examples

### Happy Path

Fetch `sale.order` with `partner_id`; many2one field becomes `{partner_id_id, partner_id_display_name}` columns.

### Error

Wrong API key returns:
```json
{"error_code":"auth_failed","message":"Authentication failed. Check your username and API key."}
```

### Edge Case

Binary/one2many/many2many fields are skipped with warnings and fetch can still succeed if at least one supported field remains.

## Mock Specification

Connection test succeeds unless `apiKey = "bad"`. Model search returns `sale.order` and `res.partner`. Fetch returns a small Odoo-source `CatalogTable`. Sync returns inserted/updated counts.
