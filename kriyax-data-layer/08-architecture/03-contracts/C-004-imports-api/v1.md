# Contract C-004: Imports API

**Version:** v1
**Status:** FROZEN
**Last updated:** 2026-05-03
**Type:** http-api
**Producer:** backend-imports-service via backend-api-routers
**Consumers:** frontend-connectors

## Purpose

Support file upload, schema inspection, editable mapping, preview, and final commit to DuckDB plus catalog registration.

## Shape

### POST `/api/imports/upload`

Request: `multipart/form-data` with `file`.

Response 200:
```json
{
  "fileName": "orders.csv",
  "filePath": "/workspace/data/uploads/id-orders.csv",
  "rowCount": 2,
  "requiresRename": false,
  "columns": [{"sourceIndex":0,"sourceName":"Customer","targetName":"customer","inferredType":"text","selectedType":"text","status":"ready"}],
  "previewRows": [{"Customer":"Acme"}]
}
```

### POST `/api/imports/commit`

Request:
```json
{"filePath":"/workspace/data/uploads/id-orders.csv","targetSchema":"raw_files","tableName":"orders","columns":[{"sourceIndex":0,"sourceName":"Customer","targetName":"customer","inferredType":"text","selectedType":"text","status":"ready"}]}
```

Response 200: `CatalogTable`

Errors:
- 400 `invalid_file`, `validation_failed`, `duplicate_column`
- 409 `table_exists`
- 500 `import_failed`

## Examples

### Happy Path

Upload `orders.csv`, edit column type, commit to `raw_files.orders`, receive a `CatalogTable`.

### Error

Duplicate target names:
```json
{"error_code":"duplicate_column","message":"Column name 'customer' is duplicated. Rename it before import.","fields":{"columns":"customer"}}
```

### Edge Case

Headers-only CSV returns `rowCount: 0` and commit creates an empty table with columns.

## Mock Specification

Upload mock accepts `.csv` and `.xlsx`; rejects other suffixes. Commit mock rejects `tableName = "exists"` with 409, otherwise returns a file-source `CatalogTable`.
