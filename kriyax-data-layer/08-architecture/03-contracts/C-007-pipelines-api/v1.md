# Contract C-007: Pipelines API

**Version:** v1
**Status:** FROZEN
**Last updated:** 2026-05-03
**Type:** http-api
**Producer:** backend-pipelines-service via backend-api-routers
**Consumers:** frontend-pipelines, frontend-app-shell

## Purpose

Create pipelines from saved scripts, attach optional connector sync pre-steps, schedule runs, trigger manually, inspect history, and surface failures.

## Shape

### POST `/api/pipelines`

Request:
```json
{"pipelineName":"Daily Clean Orders","scriptId":"clean_orders.py","connectorSyncId":null}
```

Response 200:
```json
{"id":"abc123","name":"Daily Clean Orders","scriptId":"clean_orders.py","connectorSyncId":null,"enabled":true,"schedule":null,"createdAt":"2026-05-03T00:00:00+00:00","updatedAt":"2026-05-03T00:00:00+00:00","lastRun":null}
```

### GET `/api/pipelines`

Response 200:
```json
{"pipelines":[]}
```

### PATCH `/api/pipelines/{pipelineId}/enabled`

Request:
```json
{"enabled":false}
```

Response 200: Pipeline.

### PATCH `/api/pipelines/{pipelineId}/schedule` target

Request:
```json
{"type":"daily","timeOfDay":"06:00","timezone":"Asia/Kolkata"}
```

Response 200:
```json
{"schedule":{"type":"daily","timeOfDay":"06:00","timezone":"Asia/Kolkata","nextRunAt":"2026-05-03T00:30:00+00:00","updatedAt":"2026-05-03T00:00:00+00:00"}}
```

### POST `/api/pipelines/{pipelineId}/run`

Response 200: `PipelineRun`

### GET `/api/pipelines/{pipelineId}/runs?status=all`

Response 200:
```json
{"runs":[]}
```

### GET `/api/pipelines/runs/{runId}`

Response 200: `PipelineRun`

### GET `/api/pipelines/failures` target

Response 200:
```json
{"failures":[{"pipelineId":"abc123","runId":"run123","status":"active","createdAt":"2026-05-03T00:00:00+00:00"}]}
```

### POST `/api/pipelines/failures/{runId}/ack` target

Response 200:
```json
{"acknowledged":true,"runId":"run123"}
```

Errors:
- 400 `validation_failed`, `invalid_schedule`, `invalid_cron`
- 404 `pipeline_not_found`, `run_not_found`, `script_not_found`
- 409 `pipeline_name_exists`, `already_running`
- 500 `pipeline_run_failed`

## Examples

### Happy Path

Create pipeline from `clean_orders.py`, set daily 6am schedule, manually run, and see the run in history.

### Error

Invalid cron schedule returns:
```json
{"error_code":"invalid_cron","message":"Cron expression must have five fields."}
```

### Edge Case

Disabled pipeline keeps its schedule but automatic runs are skipped; manual run remains allowed.

## Mock Specification

Create returns a pipeline with generated `id`. Schedule mock rejects cron strings with fewer than five fields. Run mock returns success unless pipeline ID starts with `fail`.
